☁️ Google Drive Sync Setup
⚠️ This is a one-time 5-minute setup. You'll need to run the app from localhost:8080 (instructions in step 6 below) — Google requires this for OAuth to work. Once set up, your data auto-saves to Google Drive every time you make a change.
1
Go to Google Cloud Console
Open console.cloud.google.com and sign in with your Google account (the one you want to save data to).
2
Create a New Project
Click the project dropdown at the top → New Project → name it Top Tier Tracker → click Create.
3
Enable Google Drive API
In the left menu go to APIs & Services → Library → search Google Drive API → click it → click Enable.
4
Create OAuth Credentials
Go to APIs & Services → Credentials → Create Credentials → OAuth client ID.

• Application type: Web application
• Name: Top Tier Tracker
• Authorized JavaScript origins: add http://localhost:8080

Click Create. A popup shows your Client ID — copy it.
5
Configure OAuth Consent Screen
Go to OAuth consent screen → choose External → fill in App name Top Tier Tracker and your email → Save. Under Test users, add your own Google email.
6
Run the app from localhost
Google requires the app to be served from http://localhost:8080 — not opened as a file. Open Terminal (Mac) or Command Prompt (Windows) and run:

cd ~/Desktop && python3 -m http.server 8080

Then open http://localhost:8080/job-tracker.html in Chrome. Keep Terminal open while you work — close it when done.
7
Paste Your Client ID Below
Copy your Client ID from step 4 and paste it here. It looks like: 123456789-abc...apps.googleusercontent.com
